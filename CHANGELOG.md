# 更新日志

## v0.1.1-mac (2026-05-18)

### macOS 平台修复
- **Electron 启动修复**：修复 `ELECTRON_RUN_AS_NODE=1` 环境变量导致 Electron 以 Node.js 模式运行的问题
- **preload 路径修复**：修复主进程加载预加载脚本的路径错误
- **notify 参数修复**：修复 `notify_task_result()` 参数名不匹配的问题

### 新增（合并自 Windows v0.1.1）
- 模块设置页面支持自定义样式 (`settings.css`)
- 字段定义支持 `width` 属性设置输入框宽度
- 后端 API: `/api/modules/{name}/style` 获取模块自定义样式
- 前端动态注入/清理模块样式

### 优化
- 模块配置独立化，每个模块管理自己的配置
- 数据与模块分离存储（`data/` 目录）
- 进入设置页面时正确加载已保存的定时配置
- 左侧导航图标映射优化，支持模块自定义图标

### 启动脚本改进
- `start.sh` 添加 `unset ELECTRON_RUN_AS_NODE` 确保正确启动
- 简化启动流程，由 Electron 主进程管理 Python 后端

## v0.1.1 (2026-05-18) - Windows 版本

### 新增
- 模块设置页面支持自定义样式 (`settings.css`)
- 字段定义支持 `width` 属性设置输入框宽度
- 推送内容显示变化的数据详情

### 优化
- 点检模块推送内容简化，只显示门店名称和月度数据
- 模块配置独立化，每个模块管理自己的门店列表
- 数据与模块分离存储（`data/` 目录）
- 进入设置页面时正确加载已保存的定时配置
- ADB 端口改为全局配置，模块不再单独配置

### 修复
- 修复定时配置加载后显示默认值的问题
- 修复调度器语法错误导致模块无法加载的问题
- 移除不需要的超时控制代码

### 改进
- 点检模块添加详细日志输出，便于诊断问题
- 模块目录结构优化

## v0.1.0 (2026-05-14)

### 初始版本
- 模块化架构：每个功能模块独立管理
- 可视化调度：任务等待中 → 排队中 → 运行中
- 灵活定时：支持每天、每周、间隔、自定义 cron
- 通知推送：支持企业微信 Webhook 和邮件通知
- 实时日志：前端实时显示任务执行日志
